Reviews — cross-repo PR review module

A new top-level module (peer of Instances / TimeTracker / Settings) that lists all open PRs across every project repo — GitHub via gh, Škoda Azure DevOps (PPS, Spot) via REST + PAT — and lets you fire a review agent at any of them. The agent checks out the PR branch in a throwaway git worktree, reviews with full repo context, and returns a markdown report + structured findings. You can also open any PR to read its diff in-app before deciding to review, and each finding links straight to its diff line. You read the report first; posting findings back to the PR is an explicit, opt-in second step.

Build order — three independently shippable sub-projects

1PR listing + diffship first

Both host adapters + normalized PR model + list UI, refresh, and the in-app Diff tab. A unified PR inbox you can read on day one — zero write risk.

2Review runnercore

Worktree checkout → headless review agent → report drawer. Persists to a pr_reviews table (migration v6).

3Post-backopt-in

Push selected findings to the PR as inline comments / a review, on GitHub or DevOps. Designed once the read side is proven.

Review runner (2): firing Review ▸ spins a throwaway git worktree at the PR branch in the local clone, runs a headless review agent with full repo context, and streams the result into this drawer as a persisted report (summary + structured findings, each with severity · category · file:line). Post-back (3): tick the findings worth sharing and push them to the PR as inline comments — host-aware (GitHub review / DevOps threads). "Copy markdown" is the always-available escape hatch.

End-to-end flow

Refresh→ gh + DevOps REST→ normalized PR list→ view diff (in-app)→ Review ▸→ worktree @ PR branch→ headless review agent→ report + findings (persisted)→ select→ post to PR (opt-in)
